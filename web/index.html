Extra blocks · NYM-EXIT abuse blocklist

    

    

      
      
      
The blocks live in the NYM-EXIT chain, which nym-node rebuilds
        on start — so a oneshot unit re-applies them after each start. Install only applies the list to the
        running chain immediately (no restart) and enables the unit; tick the box to also restart nym-node and confirm
        the full flush-and-reapply, or restart later and use Verify. Verification checks the REJECT rules are
        present in the chain (the authoritative signal — a from-node ping wouldn't traverse the exit forward path).
        IPv4 goes in iptables NYM-EXIT; IPv6 entries are applied to
        ip6tables NYM-EXIT when the node has a v6 exit chain (set
        MAESTRO_NYM_EXIT6 on the agent if yours is named differently).

    

    

      
      
      
      
    

  






  

    
Wallets · balances, rewards & transfers

    
Keys stay on this machine () and are decrypted only for the operation you run — nothing is sent to the nodes. Flow: Query operator rewards → Withdraw them into the wallet balance → Send from the balance.

    

    

      
      

        
        

          
          
        

        
Send is irreversible. The same amount is sent from each selected wallet to the receiver.
          Send max sends the floor of the balance in whole NYM and keeps the sub-1-NYM remainder for the network fee.
